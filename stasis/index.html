Stasis Save System

Production-grade save and load for Unity 6. AES-256-GCM encryption, source-generated zero-reflection serialization, schema migration, atomic writes, and a full editor toolset.

Full Documentation (PDF) Quick Start Changelog
Security: Never commit encryption keys to version control. StasisSettings stores your key — exclude it from VCS or inject at runtime via SetRuntimeEncryptionKey(). See Chapter 4 of the PDF for the recommended approach.

Quick Start

1

Add StasisManager to a GameObject in your scene and assign a StasisSettings asset (Assets → Create → RangaDigital → Stasis → Settings).

2

Mark any MonoBehaviour with [SaveSlot] — the source generator emits StasisSave and StasisLoad at compile time, zero reflection at runtime.

[SaveSlot]
public partial class PlayerStats : MonoBehaviour
{
    public int  Health = 100;
    public int  Score  = 0;
    public int  Level  = 1;
}
3

Add a StasisIdentity component to the same GameObject. This assigns a stable GUID so saves survive scene reloads and restarts.

4

Save and load from anywhere:

await StasisManager.Instance.SaveAsync("slot-1");
await StasisManager.Instance.LoadAsync("slot-1");
5

Subscribe to events for UI feedback:

StasisManager.Instance.OnSaveCompleted += slot => Debug.Log("Saved!");
StasisManager.Instance.OnLoadCompleted += slot => Debug.Log("Loaded!");

For objects not suited to the source generator, implement ISaveable directly:

public class Enemy : MonoBehaviour, ISaveable
{
    public void StasisSave(SaveSection s) { s.WriteInt32("hp", _hp); }
    public void StasisLoad(SaveSection s) { _hp = s.ReadInt32("hp", 100); }
    public MigrationMap GetMigrationMap() => MigrationMap.Empty;
}

Changelog

1.0.0 — 2026-04-07 — Initial Release

Core Pipeline

  • StasisManager singleton with async SaveAsync / LoadAsync
  • Event-driven feedback: OnSaveBegan, OnSaveCompleted, OnLoadBegan, OnLoadCompleted, OnError
  • Multi-slot save management with configurable slot names and paths
  • Auto-save with configurable interval and lifecycle hooks
  • ISaveable interface for manual save/load integration
  • [SaveSlot] attribute — Roslyn source generator, zero reflection at runtime
  • SaveSection typed API: WriteInt32, WriteFloat, WriteString, WriteVector3, WriteBool, WriteBytes and symmetrical Read* variants

Serialization

  • Source-generated serializer — no System.Reflection at runtime
  • JSON profile (human-readable), Binary profile (compact, high-performance)
  • Full Unity type support: Vector2/3/4, Quaternion, Color, Rect, Bounds, Matrix4x4, AnimationCurve, Gradient, LayerMask

Security

  • AES-256-GCM authenticated encryption (IL2CPP / standalone builds)
  • AES-CBC fallback for Unity Editor / Mono — logged as error (expected)
  • Per-save cryptographically random IV — no IV reuse across saves
  • HMAC-SHA256 save signing with TrustState verification
  • SetRuntimeEncryptionKey() for non-VCS key injection
  • StasisKeyRotationWindow — re-encrypt saves with a new key

Storage & Reliability

  • Atomic write pipeline: write-to-journal → fsync → rename, .bak rollback on crash
  • .meta sidecar for fast slot inspection without decrypting payload
  • Platform-aware storage paths with per-platform profile overrides
  • Chunked storage for large payloads; rolling backups with configurable retention
  • Cloud sync via ICloudSaveProvider + HTTP + folder-mirror backends

Migration

  • MigrationMap — declare key renames and value transforms between schema versions
  • [PreviouslySavedAs] attribute for field renames
  • SchemaFingerprint in save header detects incompatible schema changes before load

Editor Tooling

  • Save Inspector — browse save file contents without running the game
  • Build Validator — blocks Mono release builds when AES-GCM is configured
  • Key Rotation Utility — re-encrypt an entire save directory from one key to another
  • Schema Diff Window, EasySave Migration Window, Diagnostics Window

Unity Ecosystem Integrations

  • StasisAnimatorSaveable, StasisRigidbodyStateSaveable, StasisColliderStateSaveable
  • StasisCharacterControllerSaveable, StasisNavMeshAgentSaveable
  • StasisAudioSourceSaveable, StasisAudioMixerSnapshotSaveable
  • StasisParticleSystemSaveable, StasisVisualEffectSaveable
  • StasisCinemachineCameraSaveable, StasisPlayableDirectorSaveable (Timeline)
  • StasisCommonStateSaveable — zero-code Inspector-driven state saving
  • StasisVisualScriptingApi

Platform & Console

  • Bridge templates for PS5 (PlayStation Network), Xbox Series (Connected Storage), Nintendo Switch
  • WebGL IndexedDB backend with frame-budget yielding and soft-limit inspector warning

Demo & Beginner Guide

  • Three-step beginner progression: zero-code Inspector → manual ISaveable → [SaveSlot]
  • UAT prototype — primitives-only playable scene, save/load verified end-to-end
